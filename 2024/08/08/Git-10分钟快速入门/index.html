Git10分钟快速入门

Git简介

Git 是一个免费的、开源的分布式版本控制系统，可以快速高效地处理从小型到大型的各种项目。

什么是版本控制系统？

版本控制系统

  • 版本控制软件功能：
    • 版本管理：回退到历史上的任何版本
    • 共享代码：团队之间共享代码
    • 代码整合：团队合作开发
  • 版本控制是一种记录文件内容变化， 以便将来查阅特定版本修订情况的系统。

Git 易于学习， 占地面积小， 性能极快。 它具有廉价的本地库， 方便的暂存区域和多个工作流分支等特性。 其性能优于 Subversion、CVS、Perforce 和 ClearCase 等版本控制工具。

Git的特性使其特别适用于开源项目的管理，本文接下来给出git的高频命令，让大家10分钟快速上手git。

GIT命令

首先阐明几个关键概念

工作目录 (Working Directory)

工作目录 是正在进行开发的目录。它包含了所有项目的文件和子目录，是直接进行代码编辑和修改的地方。工作目录中的文件可以处于以下三种状态之一：

  1. **已修改 (Modified)**：文件已经被修改，但还没有被添加到暂存区。
  2. **已暂存 (Staged)**：文件已经被修改，并添加到了暂存区，将在下次提交中被包含。
  3. **未修改 (Unmodified)**：文件没有被修改，或者修改已经被提交到版本库。

暂存区 (Staging Area)

暂存区 (也称为“索引”或“缓存”) 是一个介于工作目录和版本库之间的区域。用来临时保存你即将提交的更改。

版本库 (Repository)

版本库 是 Git 用来保存项目的历史记录和所有提交的地方。它包含了所有已经提交的快照（commits），每个提交都是一个独立的版本。版本库通常分为两种：

  1. 本地版本库：存储在你自己计算机上的版本库。
  2. 远程版本库：存储在服务器上的版本库（例如 GitHub、GitLab），通常用于协作开发。

0x02 上传命令

git add <文件名>

在暂存区加入文件

git commit -m <评论内容>

提交更改（本地）

git push

push本地内容到远程仓库

0x03 拉取命令

git fetch

从远程仓库获取更新，不合并到工作目录

git merge

将fetch的内容合并到工作目录

git pull

从远程仓库获取更新并合并到工作目录

git pull 看起来是git fetch 和git merge的组合，

git pull 更加简便，

但git fetch 和git merge的组合分步骤的拉取内容，当其他人在远程仓库更改内容的情况下可能导致合并过程中出现冲突，此时我们就可以自定义的选择保留他人的更改还是自己本地仓库的内容以解决冲突，提供了更好的鲁棒性

0x04 日志、状态

git log

查看项目的所有提交历史，帮助我们回退版本

git status

查看当前工作目录和暂存区的状态。

未跟踪的文件（Untracked files）：
  • 这些文件是新文件，Git 还没有将其纳入版本控制。
  • 你可以使用 git add 命令将这些文件添加到暂存区。
未暂存的更改（Changes not staged for commit）：
  • 这些文件已经被跟踪，但自上次提交以来有更改，尚未被添加到暂存区。
  • 你可以使用 git add 命令将这些更改添加到暂存区。
暂存的更改（Changes to be committed）：
  • 这些文件已经被添加到暂存区，并将在下次提交时包含在内。
  • 你可以使用 git commit 命令提交这些更改。

0x05 git branch

开源最有意思的点，开新分支自己玩

git branch new-branch

创建新分支

git switch <branch-name>

切换到新分支(本地的文件夹也会切换)

git branch -d <branch-name>

删除分支

git branch -m <old-branch-name> <new-branch-name>

重命名分支

git push -u origin <new-branch>

新分支的初次推送，并自动设置跟踪关系

git branch --set-upstream-to=origin/<remote-branch>

适用于已有分支，只设置跟踪关系而不进行推送

git branch -vv

显示所有本地分支以及对应的上游分支（如果有）

0x06 fork

把其他人的 Git 仓库复制到你的 GitHub 账户下，新的仓库，这个副本是独立的
