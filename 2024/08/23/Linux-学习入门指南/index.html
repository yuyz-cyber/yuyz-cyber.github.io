Linux 学习入门指南

Created2024-08-23|Updated2024-08-23
|Post Views:

1.linux如何入门使用？

1.1 直接配置linux系统&购买服务器(新手不建议)

在有米的情况下，不妨直接配置一台LInux或购买一台服务器（此方式一般通过ssh连接，只可以执行bash指令）使用，但不建议新手这样，因为Linxu系统的使用容错性比较第，万一搞错一个环节整个机器可能就毁了。

1.2 使用虚拟化技术（wsl，虚拟机，容器）

一般建议新手先用虚拟化技术学习Linux(成本小容错高)，等熟悉之后再玩真机器。

下面介绍常见的三种虚拟化技术:

1.2.1 WSL（Windows Subsystem for Linux）

WSL 是一种允许在 Windows 上运行 Linux 二进制可执行文件的兼容层。它并不是传统的虚拟机，而是通过操作系统级别的虚拟化技术，将Linux系统嵌入到Windows系统中，目的是在 Windows 环境中支持 Linux 工作流。

  • WSL 1：基于一个兼容层，翻译 Linux 系统调用为 Windows 系统调用，从而允许 Linux 程序在 Windows 上运行。但由于这种翻译存在，某些功能和性能上不如原生 Linux 环境，有不少依赖于内核的应用是无法正常运行的——例如Docker。
  • WSL 2：基于一个轻量级的虚拟机，包含完整的 Linux 内核，性能更接近原生 Linux 系统。它还支持完整的 Linux 文件系统和 Docker 容器。

1.2.2 虚拟机（Virtual Machine）

虚拟机是一种虚拟化技术，它允许在一台物理机器上运行多个独立的操作系统实例。每个虚拟机都有自己的虚拟硬件（如 CPU、内存、磁盘等）和完整的操作系统内核。

  • Hypervisor（虚拟机管理程序）：Hypervisor 管理多个虚拟机，并将物理硬件资源分配给这些虚拟机。Hypervisor 可以是基于软件的（ VMware、VirtualBox是最常用的俩个虚拟机软件，拥有快照功能以提供高容错）或基于硬件的（如 KVM、Hyper-V）。
  • 虚拟机与宿主机：虚拟机运行在 Hypervisor 上，每个虚拟机都有独立的操作系统，与宿主机完全隔离。

1.2.3 容器（Container）

容器是一种轻量级的虚拟化技术，它将应用程序及其依赖打包到一个独立的环境中运行。与虚拟机不同，容器共享宿主机的操作系统内核，但依然通过隔离技术（如命名空间、Cgroups 等）提供独立的运行环境。

  • Docker：最流行的容器化平台，使用命名空间和控制组实现容器的隔离和资源管理。
  • Kubernetes：容器编排工具，用于管理大规模的容器部署。

2. 虚拟化技术的安全问题

“云时代”的虚拟机安全被提升到至关重要的位置。

虚拟机逃逸（Virtual Machine Escape）是指恶意程序或攻击者利用某种漏洞或技术手段突破虚拟机的限制，从虚拟机中“逃逸”出来，实现与宿主机操作系统交互的一个过程，进而在宿主机上执行代码、获取敏感数据，甚至控制宿主机或其他虚拟机，或者在宿主机上运行恶意软件。
